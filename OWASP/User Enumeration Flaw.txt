## User Enumeration Flaw

**Scenario:**  
In WeakApp, the login utility checks the username and password against the database. If either is incorrect, it displays an error message.

**Testing Steps:**

1. **Incorrect Username & Password**
    - Username: `vvg1`
    - Password: `1234`
    - Output: Error message.

2. **Correct Username & Incorrect Password**
    - Username: `vvg`
    - Password: `1234`
    - Output: Error message.

3. **Incorrect Username & Correct Password**
    - Username: `vvg1`
    - Password: `vvg!123`
    - Output: Error message.

4. **Correct Username & Password**
    - Username: `vvg`
    - Password: `vvg!123`
    - Output: Successful login.

**Risk:**  
Attackers can deduce valid usernames and build a list for brute force attacks.

**Mitigation:**
- Display a generic error: “Username or Password is invalid.”
- Track and block brute force attempts.
- Use CAPTCHA.
- Enforce strong, encrypted passwords.
